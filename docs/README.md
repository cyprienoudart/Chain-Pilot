## üí° Tips for Using Documentation

### Search Tips
- Use Ctrl+F (Cmd+F on Mac) to search within documents
- Check the Table of Contents at the top of each document
- Look for code examples highlighted in gray boxes

### Understanding Examples
- `bash` examples can be run directly in your terminal
- `json` examples show API request/response formats
- `python` examples show Python SDK usage (if applicable)

### Getting Help
If documentation is unclear or missing:
1. Check the [API interactive docs](http://localhost:8000/docs) (when server is running)
2. Review test files in `tests/` for usage examples
3. File an issue on GitHub
4. Contact: support@chainpilot.dev

## ‚ùì FAQ

**Q: Where do I start?**  
A: Begin with the [Quick Start Guide](guides/QUICKSTART.md), then explore the [API Documentation](API.md).

**Q: Is this production-ready?**  
A: Yes! ChainPilot v1.0.0 is production-ready with 100% test coverage. See [Security Documentation](phases/PHASE6_SECURITY.md) for deployment best practices.

**Q: How do I test my changes?**  
A: Follow the [Testing Guide](guides/TESTING_GUIDE.md) to run all test suites.

**Q: Can I use this with mainnet?**  
A: Yes, but start with testnet (Sepolia) first. See [Security Documentation](phases/PHASE6_SECURITY.md) for production considerations.

**Q: Where's the API documentation?**  
A: See [API.md](API.md) for complete reference, or visit http://localhost:8000/docs for interactive docs.

**Q: How do I configure rules?**  
A: Check the [Rule Engine Documentation](phases/HOW_PHASE3_WORKS.md) for examples and best practices.

**Q: Is there a demo?**  
A: Yes! Follow the [Demo Guide](guides/DEMO_GUIDE.md) for step-by-step instructions.

**Q: How do I contribute?**  
A: See the contributing section above, or refer to the main [README](../README.md).
